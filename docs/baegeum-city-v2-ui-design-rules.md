# 배금도시 v2 UI 디자인 규칙

결론: 배금도시 v2 초기 UI는 화면을 막는 큰 개발 패널을 피하고, `돈/상태 HUD`, `미니맵`, `휴대폰`, `설정`, `가방`, `채팅`, `모바일 액션`이 서로 겹치지 않게 작은 레이어로 나눈다.

## 현재 규칙

- 중앙 상단 월드 시간은 숨기고, 시간/일차는 휴대폰과 내부 시스템에서만 사용한다.
- 왼쪽 상단에는 `DAY`, 현금, 계좌, 칩을 둔다.
- 오른쪽 상단에는 체력, 에너지, 배고픔을 두되 미니맵을 덮지 않도록 왼쪽으로 뺀다.
- 오른쪽에는 설정 버튼과 가방 버튼을 세로로 분리한다.
- 디버그 정보, 조작법, 에디터 링크, 테스트장 링크는 설정 패널 안에 숨긴다.
- 미니맵은 오른쪽 상단에 둔다.
- 휴대폰은 오른쪽 하단에 둔다.
- 모바일에서는 `ACTION` 버튼이 PC의 `E`와 같은 입력 경로를 사용한다.
- 기본 화면에서는 플레이어 주변 월드가 최대한 많이 보여야 한다.

## 금지

- 왼쪽 상단에 큰 개발자 카드 고정
- 여러 개의 큰 카드형 HUD를 동시에 노출
- 디자인 확인 전에 기능 정보를 계속 화면 위에 얹기
- “AI가 만든 웹 대시보드”처럼 보이는 과한 패널/그라데이션/설명문

## 허용

- 설정 버튼으로 접히는 개발 정보
- 작은 시간 배지
- 작은 미니맵
- 필요할 때만 열리는 휴대폰
- MammonCity 원본 id/class 기반의 작은 HUD와 가방 패널
- PC `E`와 같은 규칙을 쓰는 모바일 액션 버튼
- 임시 디버그 정보는 설정 패널 안에 넣기
- 테이블에 앉았을 때만 나타나는 작은 미니게임 패널. 기본 이동 중에는 숨긴다.
- 도박장 안에서만 열리는 DiceLand ATM형 환전 패널. 기본 이동 중이나 휴대폰 포커스 중에는 숨긴다.

## 다음 UI 작업 기준

새 HUD를 추가하기 전에 먼저 물어본다.

```text
이 정보가 기본 화면에 항상 보여야 하는가?
아니면 설정/폰/미니맵 안으로 들어가도 되는가?
```

답이 애매하면 기본 화면에 추가하지 않는다.
