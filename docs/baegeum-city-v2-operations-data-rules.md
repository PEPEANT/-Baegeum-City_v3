# 배금도시 v2 운영/데이터/확장성 규칙

결론: 배금도시 v2는 온라인 도박 도시가 되기 전에 `ID`, `저장`, `권한`, `검증`, `재접속`, `서버 시간`, `감사 로그`, `마이그레이션`, `가상 재화` 규칙을 먼저 고정한다.

## ID 규칙

모든 영속 데이터는 사람이 읽을 수 있고 충돌이 적은 ID를 사용한다.

```text
player:p_123
npc:city-main:guard-001
venue:blackjack-casino-01
door:blackjack-casino-01:front
interior:blackjack-casino-01
table:blackjack-casino-01:main
round:odd-even-casino-01:20260525:000001
ledger:p_123:20260525:000001
audit:admin-001:20260525:000001
```

규칙:

- UI 표시 이름과 시스템 ID를 섞지 않는다.
- 한국어 간판은 `sign`, 시스템 식별자는 `id`에 둔다.
- 서버가 만드는 ID와 에디터가 만드는 ID의 prefix를 분리한다.
- 라운드/ledger/audit ID는 서버 시간 기준으로 만든다.

## 저장/세이브 구조

저장소는 목적별로 분리한다.

- `localDraft`: 에디터 임시 저장, 개인 브라우저에만 존재
- `publishedMap`: 온라인 서버가 승인한 운영 맵
- `playerSave`: 플레이어 계정/위치/재화
- `economyLedger`: 돈/칩/베팅/승패 기록
- `auditLog`: 관리자/운영 기록

금지:

- 에디터 draft를 바로 온라인 운영 맵으로 사용하지 않는다.
- 클라이언트 localStorage 값을 서버 재화로 신뢰하지 않는다.
- 플레이어 저장과 맵 저장을 같은 객체에 섞지 않는다.

## 권한 모델

권한은 최소 5단계로 나눈다.

- `player`: 일반 플레이어
- `spectator`: 관전자, 베팅 불가
- `moderator`: 채팅 제재, 신고 처리, 관찰 가능
- `admin`: 강제 이동, 강제 퇴장, 방 상태 확인 가능
- `owner`: 맵 publish, 경제 보정, 권한 부여 가능

규칙:

- 권한은 클라이언트 UI가 아니라 서버 세션에서 판정한다.
- `admin`과 `owner` 행동은 모두 `auditLog`에 남긴다.
- 관리자 관전은 일반 랭킹/플레이어 수 집계에 들어가지 않는다.

## 안티치트/검증 규칙

서버는 최소한 아래를 거부한다.

- 비정상 위치 순간이동
- 벽/건물 내부 좌표 이동
- 너무 빠른 위치 업데이트
- 음수 베팅
- 잔고보다 큰 베팅
- 같은 라운드 중복 베팅
- 마감 시간 이후 베팅
- 관전자/관리자 모드의 베팅 요청
- 클라이언트가 직접 보낸 승패 결과

첫 구현은 완벽한 안티치트보다 “서버가 돈과 결과를 확정한다”는 원칙을 지키는 것이 우선이다.

## 재접속 처리

연결이 끊긴 플레이어는 즉시 삭제하지 않는다.

```text
connected -> disconnected_grace -> reconnected
connected -> disconnected_grace -> expired
```

규칙:

- 짧은 유예 시간 동안 좌석과 상태를 보존한다.
- 라운드 진행 중 연결이 끊겨도 이미 수락된 베팅은 취소하지 않는다.
- 재접속하면 서버가 마지막 승인 상태를 내려준다.
- 유예 시간이 지나면 테이블 좌석은 비우되 ledger 기록은 유지한다.

## 시간/틱 기준

게임 시간은 서버 기준이다.

- 라운드 시작/마감
- 경마 출발/도착
- 랭킹 집계
- 채팅 rate limit
- 재접속 유예
- 일일 보상/랭킹 초기화

클라이언트 시간은 UI 카운트다운 표시용으로만 사용한다.

## 감사 로그

아래 행동은 반드시 `auditLog`에 남긴다.

- 관리자 강제 이동
- 관리자 강제 퇴장
- 채팅 mute/ban
- 환불
- 베팅 취소
- 칩 지급/차감
- 맵 publish
- 권한 부여/회수

감사 로그는 수정하지 않고 새 기록으로 정정한다.

## 데이터 마이그레이션

맵/경제/도박장 스키마가 바뀌면 버전을 올린다.

```js
{
  mapVersion: "baegeum-city-v2-map-001",
  venueSchemaVersion: "venue-schema-001",
  economyVersion: "economy-001",
  saveVersion: "save-001"
}
```

규칙:

- 구버전 저장 데이터를 읽을 때 변환 함수를 둔다.
- 변환 실패 시 플레이어 재화는 보존하고 위치만 안전 스폰으로 돌린다.
- 경제 밸런스 변경은 ledger 원본을 수정하지 않는다.

## 가상 재화/도박 표현 수위

배금도시 v2의 도박은 게임 내 가상 재화로만 동작한다.

금지:

- 실제 현금 입금
- 실제 현금 출금
- 암호화폐/포인트 환전
- 외부 결제와 도박 재화 연결

규칙:

- UI에는 게임 내 재화임을 명확히 표시한다.
- 서버 데이터 모델에서도 실제 화폐 필드를 만들지 않는다.
- 운영 로그와 ledger는 게임 내 재화 단위만 기록한다.

## 구현 게이트

온라인 서버를 만들기 전 확인할 것:

- ID prefix 목록이 문서와 코드에 일치한다.
- 에디터 draft와 published map 저장소가 분리되어 있다.
- 권한별 허용 이벤트 표가 있다.
- 재접속 유예 상태가 `playerState`에 들어갈 수 있다.
- 서버 시간 기준 라운드/채팅 제한 설계가 있다.
- 관리자 행동은 audit 대상인지 먼저 확인한다.
