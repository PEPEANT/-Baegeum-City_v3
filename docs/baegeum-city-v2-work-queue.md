# 배금도시 v2 작업 큐

결론: 여러 채팅에서 동시에 개발할 때는 **맵에디터 트랙, 인게임 트랙, 온라인/경제 트랙을 분리하고, 서로 만나는 데이터 계약만 먼저 고정**한다.

## 목적

지금 배금도시는 맵에디터, 인게임, 경제, 도박장, 가상 인터넷이 동시에 커지고 있다. 한 채팅에서 건물 배치를 바꾸고 다른 채팅에서 입장/칩교환소를 만들면 같은 파일과 상태를 건드리기 쉽다. 그래서 당장 새 기능이 떠오르지 않을 때는 아래 작업 큐에서 한 줄씩 꺼내 진행한다.

## 맵에디터 트랙

맵에디터는 현재 인게임 기능을 직접 완성하기보다, 도시를 안정적으로 만들고 검증하는 도구가 된다.

1. 실행취소/되돌리기와 저장 전 변경점 표시는 현재 구현되어 있다.
2. 드래프트 저장의 `schemaVersion`, `draftVersion`, `mapVersion`, `savedAt`, `editorRevision`은 현재 구현되어 있다.
3. 새 오브젝트 ID prefix와 기존 ID 없는 장애물 자동 정규화는 현재 구현되어 있다.
4. 저장 전 검증 리포트는 현재 구현되어 있다.
5. 검증 리포트 항목 클릭 이동은 현재 구현되어 있다.
6. 선택 액션바의 좌표 잠금과 원위치 복귀는 현재 구현되어 있다.
7. 선택 액션바의 레이어 잠금은 현재 구현되어 있다.
8. 건설 목록은 자주 쓰는 프리셋을 상단에 고정하며, 현재 구현되어 있다.
9. 건설 목록 카테고리 접기는 현재 구현되어 있다.
10. 건물 카드 v0는 `빈 상가`, `빈 카지노`, `골목 상가` 배치 전용으로 구현되어 있고, `building_shell` 크기 편집은 소형/기본/대형 순환까지 가능하다. 이름/색 편집은 placement-only `shellName`/`shellColor`만 저장하며, 입장/실내/경제는 인게임 트랙 계약을 따른다.
11. 도로 편집은 이동 다음에 폭, 타입, 교차로 모양 순서로 확장한다.

## 인게임 트랙

인게임은 에디터가 만든 데이터를 읽어 실제 플레이 감각을 확인하는 쪽이다.

1. 에디터 드래프트가 켜졌을 때 맵 버전과 적용 상태를 HUD 또는 디버그 패널에서 확인한다.
2. 플레이어 충돌이 에디터의 새 오브젝트 충돌 범위를 따르는지 확인한다.
3. 건물 입장 상태를 `city`, `venue_lobby`, `table_seated`, `spectating`, `admin_observing`으로 유지한다.
4. 도박장/칩교환소는 맵 배치와 별개로 `ledger` entry만 만든다.
5. 차량은 처음부터 물리 엔진으로 가지 않고, 간단한 이동/충돌 확인부터 시작한다.

## 온라인/경제 트랙

온라인과 돈은 나중에 서버 권위로 바뀔 부분이라 UI보다 데이터 기록이 먼저다.

1. 돈, 칩, 아이템 변화는 `ledger`를 통하지 않으면 확정하지 않는다.
2. 베팅 UI는 결과를 계산하지 않고 요청만 만든다.
3. 랭킹은 현재 잔고가 아니라 ledger 집계 기준으로 설계한다.
4. 맵 데이터에는 `mapVersion`을 두고, 온라인 접속 시 버전 불일치를 막는다.
5. 파괴 가능한 오브젝트는 원본 맵 데이터와 런타임 상태를 분리한다.

## 지금 Codex에게 시키기 좋은 일

- "맵에디터 변경점 상세 비교를 붙여줘"
- "건물 카드 종류를 늘리되 배치만 가능하게 만들어줘"
- "에디터에서 충돌 레이어가 실제 인게임 충돌과 맞는지 검증해줘"
- "칩교환소 만들기 전에 ledger 기준으로 UI 흐름만 잡아줘"

## 현재 우선순위

1. venue 메타데이터 편집: 기존 `building_shell`의 `shellName`/`shellColor` 외형 편집과 `venue_anchor` 계약 경계를 유지한다.
2. 맵-인게임 계약: road/entrance/venue ID, 충돌/파괴 상태 분리.
3. 칩교환소: `ledger` 기반 교환만 먼저 구현.
4. 차량/파괴: 간단한 차량 이동 뒤 파괴 가능한 오브젝트와 연결.
5. 온라인/경제: 서버 권위 전환 전 ledger와 mapVersion 계약 유지.
